# Design Document: CHANGE-00d — Seed Data + Tests Base

## 1. Architecture Decisions

### AD-01: Model Base Class Resolution

**Status**: Decided

**Problem**: The codebase has two different model bases:
- `app/database.py` creates `Base = declarative_base()` → owns `Base.metadata`
- `app/models/mixins.py` uses `from sqlmodel import SQLModel` → owns `SQLModel.metadata`

New models inherit from `TimestampMixin(SQLModel)` / `BaseModel(SQLModel)`, so they register with `SQLModel.metadata`. But Alembic's `env.py` uses `target_metadata = Base.metadata`, which means Alembic will NOT discover the new models.

**Resolution**: Update `env.py` to use `SQLModel.metadata` as `target_metadata`, and update `tests/conftest.py` to use `SQLModel.metadata.create_all()` for test table creation. The `database.py` keeps `declarative_base()` as-is (it doesn't conflict — models simply use a different registry).

**Changes required**:
1. `migrations/env.py`: `target_metadata = SQLModel.metadata` (import `SQLModel` from `sqlmodel`)
2. `tests/conftest.py`: Use `SQLModel.metadata.create_all()` instead of `Base.metadata.create_all()`

**Why not merge bases?** SQLModel and declarative_base use separate registries. Merging them would require significant rework and doesn't provide benefits — they just need to point Alembic at the right metadata.

---

### AD-02: Seed Script — Async via asyncpg, not synchronous

**Status**: Decided

**Decision**: The seed script SHALL use async SQLAlchemy sessions (consistent with the existing codebase), NOT a separate sync connection. It SHALL use `async with async_session_factory() as session:` and execute raw SQL via `session.execute()` with `ON CONFLICT (codigo) DO NOTHING` for PostgreSQL.

**Rationale**: The entire codebase is async. Introducing a sync connection just for seeding adds complexity, another engine, and potential connection pool conflicts.

**Dialect handling for SQLite tests**: The seed script SHALL detect the database dialect (`session.bind.dialect.name`) and use `ON CONFLICT` for PostgreSQL vs a try/except approach for SQLite. Alternatively, tests can directly insert seed data via repositories instead of invoking the seed script.

---

### AD-03: Models use `table=True` with explicit `__tablename__`

**Status**: Decided

All domain models SHALL use `table=True` and explicit `__tablename__` (Spanish plural snake_case per Integrador.txt conventions). The mixins (`TimestampMixin`, `SoftDeleteMixin`) are non-table mixins (no `table=True`) since they're meant to be inherited.

---

### AD-04: Tests use an isolated TestModel, not domain models

**Status**: Decided

**Decision**: Unit tests for `BaseRepository[T]` SHALL use a minimal `TestModel(BaseModel, table=True)` defined within the test module. This avoids coupling tests to domain models (Rol, Usuario, etc.) which may change later.

---

### AD-05: Migration is handwritten, not autogenerated

**Status**: Decided

Migration `002_add_seed_models.py` SHALL be handwritten using `sa.Table()` and `op.create_table()` / `op.create_foreign_key()` / `op.create_index()`. Reason: the placeholder `001_initial.py` has no tables, so autogenerate would detect everything as "new" but also produce noise. A handwritten migration gives us precise control over column types, constraints, and ordering.

## 2. Model Definitions

### 2.1 Rol (table: `roles`)

```python
class Rol(TimestampMixin, table=True):
    __tablename__ = "roles"

    codigo: str = Field(max_length=20, primary_key=True)       # PK, semantic
    descripcion: str = Field(max_length=100, nullable=False)
```

- PK: `codigo` VARCHAR(20) — semantic code stored as-is
- No autoincrement — PK is manually assigned
- Inherits `created_at`, `updated_at` from TimestampMixin

### 2.2 EstadoPedido (table: `estados_pedido`)

```python
class EstadoPedido(TimestampMixin, table=True):
    __tablename__ = "estados_pedido"

    codigo: str = Field(max_length=20, primary_key=True)       # PK, semantic
    descripcion: str = Field(max_length=100, nullable=False)
    orden: int = Field(nullable=False)                          # FSM ordering
    es_terminal: bool = Field(default=False, nullable=False)    # Terminal state flag
```

- PK: `codigo` VARCHAR(20) — PENDIENTE, CONFIRMADO, EN_PREP, EN_CAMINO, ENTREGADO, CANCELADO
- `orden` INTEGER — defines FSM transition sequence (1-6)
- `es_terminal` BOOLEAN — ENTREGADO (5) and CANCELADO (6) are terminal

### 2.3 FormaPago (table: `formas_pago`)

```python
class FormaPago(TimestampMixin, table=True):
    __tablename__ = "formas_pago"

    codigo: str = Field(max_length=20, primary_key=True)       # PK, semantic
    descripcion: str = Field(max_length=100, nullable=False)
    habilitado: bool = Field(default=True, nullable=False)      # Can disable without delete
```

- PK: `codigo` VARCHAR(20) — MERCADOPAGO, EFECTIVO, TRANSFERENCIA
- All three start as `habilitado = true`

### 2.4 Usuario (table: `usuarios`)

```python
class Usuario(BaseModel, table=True):  # BaseModel = TimestampMixin + SoftDeleteMixin
    __tablename__ = "usuarios"

    id: Optional[int] = Field(default=None, primary_key=True)   # BIGSERIAL autoincrement
    email: str = Field(max_length=254, unique=True, nullable=False, index=True)
    password_hash: str = Field(max_length=60, nullable=False)    # bcrypt: CHAR(60)
    nombre: str = Field(max_length=100, nullable=False)
    apellido: str = Field(max_length=100, nullable=False)
    telefono: Optional[str] = Field(max_length=20, default=None, nullable=True)
    activo: bool = Field(default=True, nullable=False)
```

- PK: `id` BIGSERIAL autoincrement (only model with integer PK)
- `email` UNIQUE + INDEX for login lookups
- `password_hash` CHAR(60) — bcrypt outputs exactly 60 characters with cost ≥ 12
- `telefono` nullable — not all users may provide it
- `activo` BOOLEAN — soft toggle (separate from soft delete)
- Inherits `deleted_at` from SoftDeleteMixin for logical deletion

### 2.5 UsuarioRol (table: `usuarios_roles`)

```python
class UsuarioRol(TimestampMixin, table=True):
    __tablename__ = "usuarios_roles"

    usuario_id: int = Field(
        foreign_key="usuarios.id",
        primary_key=True,            # Part of composite PK
        ondelete="CASCADE"
    )
    rol_codigo: str = Field(
        max_length=20,
        foreign_key="roles.codigo",
        primary_key=True             # Part of composite PK
    )
    asignado_por_id: Optional[int] = Field(
        foreign_key="usuarios.id",
        default=None,
        nullable=True
    )
```

- Composite PK: (`usuario_id`, `rol_codigo`)
- Both FK fields are part of the composite primary key
- `usuario_id` FK → `usuarios.id` with ON DELETE CASCADE (deleting user removes role assignments)
- `rol_codigo` FK → `roles.codigo` (semantic PK)
- `asignado_por_id` FK → `usuarios.id` nullable — tracks who assigned the role (NULL for self-registered users)

## 3. Seed Script Design

### 3.1 File Structure

```
backend/app/db/
├── __init__.py          # Empty init
└── seed.py              # Seed script
```

### 3.2 Execution Model

- **Async**: Uses `asyncio.run(main())` at module level, invoked via `python -m app.db.seed`
- **Session**: Creates a new async session from the existing `async_session_factory`
- **Engine**: Uses the existing async engine from `app.database`

### 3.3 Insert Strategy

- **PostgreSQL**: `INSERT INTO ... ON CONFLICT (codigo) DO NOTHING` for models with semantic PKs
- **Usuario**: `INSERT INTO ... ON CONFLICT (email) DO NOTHING` (email is UNIQUE)
- **UsuarioRol**: `INSERT INTO ... ON CONFLICT (usuario_id, rol_codigo) DO NOTHING` (composite PK)

### 3.4 Insert Order

```python
# Order matters for FK dependencies:
1. Rol              # No FK deps
2. EstadoPedido     # No FK deps
3. FormaPago        # No FK deps
4. Usuario          # No FK deps to seed data
5. UsuarioRol       # FK to Usuario and Rol — both must exist first
```

### 3.5 Admin Password

```python
import os
from passlib.context import CryptContext

pwd_context = CryptContext(schemes=["bcrypt"], deprecated="auto")
admin_password = os.getenv("SEED_ADMIN_PASSWORD", "Admin1234!")
password_hash = pwd_context.hash(admin_password)  # cost=12 (default for passlib bcrypt)
```

| Table | Records |
|-------|---------|

| `estados_pedido` | `('PENDIENTE', 'Pedido creado, pago pendiente', 1, false), ('CONFIRMADO', 'Pago procesado y confirmado', 2, false), ('EN_PREP', 'En preparación en cocina', 3, false), ('EN_CAMINO', 'Despachado al cliente', 4, false), ('ENTREGADO', 'Entrega confirmada', 5, true), ('CANCELADO', 'Pedido cancelado', 6, true)` |

### 3.7 Error Handling

- Wrap entire seed in a try/except block
- Print success message with row counts on completion
- Print error message with traceback on failure, exit with code 1

### 4.2 Upgrade: Table Creation Order

```python
def upgrade():
    # 1. Rol (no FK deps)
    op.create_table('roles',
        sa.Column('codigo', sa.String(20), primary_key=True),
        sa.Column('descripcion', sa.String(100), nullable=False),
        sa.Column('created_at', sa.DateTime(timezone=True), server_default=sa.func.now(), nullable=False),
        sa.Column('updated_at', sa.DateTime(timezone=True), server_default=sa.func.now(), nullable=False),
    )

    # 2. EstadoPedido (no FK deps)
    op.create_table('estados_pedido',
        sa.Column('codigo', sa.String(20), primary_key=True),
        sa.Column('descripcion', sa.String(100), nullable=False),
        sa.Column('orden', sa.Integer(), nullable=False),
        sa.Column('es_terminal', sa.Boolean(), server_default='false', nullable=False),
        sa.Column('created_at', sa.DateTime(timezone=True), server_default=sa.func.now(), nullable=False),
        sa.Column('updated_at', sa.DateTime(timezone=True), server_default=sa.func.now(), nullable=False),
    )

    # 3. FormaPago (no FK deps)
    op.create_table('formas_pago',
        sa.Column('codigo', sa.String(20), primary_key=True),
        sa.Column('descripcion', sa.String(100), nullable=False),
        sa.Column('habilitado', sa.Boolean(), server_default='true', nullable=False),
        sa.Column('created_at', sa.DateTime(timezone=True), server_default=sa.func.now(), nullable=False),
        sa.Column('updated_at', sa.DateTime(timezone=True), server_default=sa.func.now(), nullable=False),
    )

    # 4. Usuario (no FK deps to seed-able tables, FK to self is only on UsuarioRol)
    op.create_table('usuarios',
        sa.Column('id', sa.BigInteger(), primary_key=True, autoincrement=True),
        sa.Column('email', sa.String(254), nullable=False),
        sa.Column('password_hash', sa.String(60), nullable=False),
        sa.Column('nombre', sa.String(100), nullable=False),
        sa.Column('apellido', sa.String(100), nullable=False),
        sa.Column('telefono', sa.String(20), nullable=True),
        sa.Column('activo', sa.Boolean(), server_default='true', nullable=False),
        sa.Column('created_at', sa.DateTime(timezone=True), server_default=sa.func.now(), nullable=False),
        sa.Column('updated_at', sa.DateTime(timezone=True), server_default=sa.func.now(), nullable=False),
        sa.Column('deleted_at', sa.DateTime(timezone=True), nullable=True),
    )

    # Email unique constraint (needs a name for clean downgrade)
    op.create_unique_constraint('uq_usuarios_email', 'usuarios', ['email'])
    op.create_index('ix_usuarios_email', 'usuarios', ['email'])

    # 5. UsuarioRol (FK to Usuario and Rol)
    op.create_table('usuarios_roles',
        sa.Column('usuario_id', sa.BigInteger(), nullable=False),
        sa.Column('rol_codigo', sa.String(20), nullable=False),
        sa.Column('asignado_por_id', sa.BigInteger(), nullable=True),
        sa.Column('created_at', sa.DateTime(timezone=True), server_default=sa.func.now(), nullable=False),
        sa.Column('updated_at', sa.DateTime(timezone=True), server_default=sa.func.now(), nullable=False),
    )

    # Composite PK
    op.create_primary_key('pk_usuarios_roles', 'usuarios_roles', ['usuario_id', 'rol_codigo'])

    # Foreign keys
    op.create_foreign_key('fk_usuarios_roles_usuario', 'usuarios_roles', 'usuarios', ['usuario_id'], ['id'], ondelete='CASCADE')
    op.create_foreign_key('fk_usuarios_roles_rol', 'usuarios_roles', 'roles', ['rol_codigo'], ['codigo'])
    op.create_foreign_key('fk_usuarios_roles_asignado_por', 'usuarios_roles', 'usuarios', ['asignado_por_id'], ['id'])
```

### 4.3 Downgrade: Reverse Order

```python
def downgrade():
    # Drop tables in reverse dependency order
    op.drop_table('usuarios_roles')
    op.drop_index('ix_usuarios_email', table_name='usuarios')
    op.drop_constraint('uq_usuarios_email', 'usuarios', type_='unique')
    op.drop_table('usuarios')
    op.drop_table('formas_pago')
    op.drop_table('estados_pedido')
    op.drop_table('roles')
```

## 6. Risks and Mitigations

| Risk | Likelihood | Impact | Mitigation |
|------|------------|--------|------------|
| SQLModel vs declarative_base metadata mismatch | High (it's a bug in existing code) | High — Alembic would create 0 tables | Resolve via AD-01: point Alembic and tests at `SQLModel.metadata` |
| ON CONFLICT syntax fails on SQLite | Medium | Medium — seed tests won't work in-memory | Detect dialect in seed script; in tests, seed via repositories instead of raw SQL |
| Admin password bcrypt hash differs between runs | Low | Low — test assertion fails | Use `pwd_context.verify()` instead of hash comparison |
| Migration 002 conflicts if 001 is modified | Low | Medium — requires manual rebase | Lock 001 as-is; 002 always revises `001_initial` |
| Test model `TestModel` conflicts with real tables | Low | Low — uses unique `test_models` table name | Isolate with unique `__tablename__`; cleanup in fixture teardown |
